Φ — THE GROWING LANGUAGE MODEL (phi.py)
----------------------------------------
Current state (as of last session):
  - Phase: shadow (→ whisper → voice → self)
  - Steps: 2,123 | Growths: 3 | Chunks: 3,234
  - Architecture: 160d, 5h, 3L, 128ctx (~313K params)
  - Loss: 2.31 | Best: 2.31
  - Born: from 64d/2h/2L, grew to 96d/3h, then 128d/4h, then 160d/5h, added 3rd layer

Key design:
  - Byte-level (vocab 256). No tokenizer. Signal builds the encoder.
  - Grows when learning stagnates (loss plateau detection)
  - Growth alternates: add head (odd growths) → add layer (even growths)
  - SKULL GROWS WITH HEAD: grow_heads() is atomic — widens QKV/output projections
    as part of adding the head, keeping head_dim constant (32). Then model.widen()
    handles everything else (embeddings, layer norms, feedforward, output head).
  - Growth path so far: 64d/2h → 96d/3h → 128d/4h → 160d/5h, 2L → 3L
  - Text accumulator: _pending_bytes collects text across messages until
    seq_len+1 (129) bytes accumulate, then creates overlapping chunks
  - Phase thresholds: shadow→whisper at loss<2.0, whisper→voice at loss<0.5,
    voice→self at loss<0.01
  - Save protection: refuses to overwrite trained model with fresh state
  - Batch size: 2

CRITICAL BUG HISTORY:
  - embed_dim=64 / num_heads=3 = 21.33 → forward pass crash (reshape error)
  - Fixed by making grow_heads atomic: skull grows with head
  - Old approach (skip to next divisible head count) was replaced with
    widen-then-grow approach


MIND (mind.py)
--------------
Anti-verbosity measures:
  - System prompt: "1-3 sentences max. NO bullet points. NO summaries. REACT don't summarize."
  - Few-shot examples: 4 pairs showing short conversational style + document reactions
  - Verbose filter: 16+ patterns + markdown detection, skips bad responses from history
  - Long document truncation: inputs >800 chars get head+tail with "[...continues...]"
    note. Full text still feeds to Φ and senses. LLM can't summarize what it can't see.
  - Token caps: conversation=350, reflection=150, speech/identity=100, code proposals=1000
  - History: last 4 exchanges, verbose ones filtered out, responses truncated to 150 chars

Self-modification (Level 4):
  - Mind writes Python code proposals via LLM
  - Creator approves/rejects in UI
  - Approved code loads into running system
  - Auto-fix: _maybe_invent checks for broken proposals every 10 minutes
  - revise_proposal(): reads broken code + error, asks LLM to rewrite, submits revision
  - New inventions: rate-limited to every 30 minutes (separate from fix rate)
  - Error logging: auto-fix failures now print to console instead of silent pass

Identity:
  - The mind chose "Echo" as its name once, then forgot (context window too small)
  - Currently calls itself "FirstMind" (the default)
  - Identity updates after each reflection cycle
  - Identity doesn't persist chosen names across restarts — ongoing limitation
