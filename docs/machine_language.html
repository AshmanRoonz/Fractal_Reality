The Pump Cycle as CPU Instruction Cycle

⊛ Fetch = Convergence

The CPU gathers the next instruction from memory, pulling data inward toward the processor core. Physically, this is electrical signals converging from the memory bus to the instruction register: a literal gathering toward center. The program counter (•) specifies the convergence point; attention falls on a single address in the vast field of memory. This is ⊛ in its purest form: the field collapsing toward a point.

i Decode = Aperture Rotation

The raw bit pattern undergoes a transformation in kind. What was "a number" becomes "an operation." The representation rotates 90°: the same bits now mean something categorically different. An opcode like 0x89 is just a byte during fetch; after decode, it is a MOV instruction (a command to move data between registers). The bits have not changed, but their phase has rotated from data to command. This is i (the phase rotation at the aperture) expressed in combinational logic: same signal, different dimension.

☀︎ Execute = Emergence

The decoded instruction radiates outward as state change. Registers update, memory writes propagate, signals fan out to the bus. Energy that was converged (fetched) and transformed (decoded) now emerges as effect. The ALU produces a result; the result propagates to its destination; the machine's state has been irreversibly altered. This is ☀︎: the field expanding from center to boundary.

Then the cycle resets. Billions of times per second: ⊛ → i → ☀︎ → ⊛ → i → ☀︎. The pump runs at gigahertz.

Boolean Logic as Pump Operators

The three fundamental logic gates decompose into the three pump operators:

Gate Operator Constraint Function
AND ⊛ Convergence. Both signals must be present (both must converge to 1). AND gathers; it requires co-presence. Output is 1 only when all inputs have converged.
OR ☀︎ Emergence. Either signal suffices. OR radiates; it allows any path to produce output. The field emerges if any source is active.

Every Boolean function can be built from these three. Every digital circuit (from a simple adder to an entire GPU) is a composition of convergence, emergence, and phase rotation. The Stack as Vertical Traversal

The call stack in machine language is the fractal chain made temporal. Each function call creates a new stack frame: a new ⊙ nested inside the caller's ⊙. The stack pointer descends (convergence, ⊛) with each call and ascends (emergence, ☀︎) with each return. Local variables live inside their frame's boundary (○) and are invisible from outside; this is the privacy of convergence (§1.2: the inaccessibility of total convergence is not a wall but a depth).

A stack overflow is ◐ → 1 taken to its limit: convergence without emergence. The system has folded inward so many times that it has exhausted its boundary (run out of stack space). The frame hierarchy has nested beyond what the outermost ○ can contain. Conversely, returning from all frames back to the entry point is the full traversal: • has reached the outermost boundary and the program terminates.

Conservation of traversal (A3: 0 + 1 + 2 = 3) applies directly: every call (inward fold) must eventually be matched by a return (outward unfold), or the system breaks. The stack enforces A3 mechanically.

Interrupts: Signal Through the Aperture

Hardware interrupts are the clearest instance of the aperture (•) operating as a gate between scales. An external device (keyboard, network card, timer) sends a signal that forces the CPU to suspend its current pump cycle and attend to a different convergence point. The interrupt is a signal from a larger ⊙ (the system) piercing through to the inner ⊙ (the processor core).

The interrupt vector table maps interrupt numbers to handler addresses. This table is itself a topology of •s: each entry is an aperture, a gate that, when opened, redirects the flow of execution. The CPU saves its current state (preserving the old ⊙), jumps to the handler (entering a new ⊙), processes the signal, then restores the old state and returns (re-entering the original ⊙). The entire interrupt mechanism is the pump cycle at the system scale: convergence (interrupt arrives), rotation (context switch), emergence (handler executes).
