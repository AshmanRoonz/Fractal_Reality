0. Abstract

The standard LSTM forget gate destroys information: it multiplies the cell state element-wise by a value between 0 and 1, permanently erasing whatever is scaled down. This paper proposes a replacement derived from the Circumpunct Framework (Roonz, 2024): instead of erasure, old state is compressed inward using a fractal exponent (D = 1.5, the balance dimension), preserving phase while reducing magnitude. The compressed state accumulates in a deep memory (analogous to the worldline i(t)) that influences future processing through resonance-gated retrieval (transmission T = cos²(Δφ/2)). Nothing is ever destroyed. Retrieval cost depends on frequency matching, not brute-force expansion.

The resulting architecture (the Fractal-LSTM) makes three modifications to standard LSTMs, each corresponding to one of the three constraints of the circumpunct: convergence (•), mediation (Φ), and emergence (☼). We describe the architecture, derive its properties from first principles, compare it to standard LSTM and transformer attention, and outline experiments for empirical validation.

1. The Problem with Forgetting

The Standard LSTM Cell

The Long Short-Term Memory network (Hochreiter & Schmidhuber, 1997; Gers et al., 2000) maintains a cell state ct that flows through time, modified at each step by three gates:

# Standard LSTM equations ft = σ(Wf · [ht-1, xt] + bf) # forget gate it = σ(Wi · [ht-1, xt] + bi) # input gate c̃t = tanh(Wc · [ht-1, xt] + bc) # candidate ct = ft ⊙ ct-1 + it ⊙ c̃t # cell state update ot = σ(Wo · [ht-1, xt] + bo) # output gate ht = ot ⊙ tanh(ct) # hidden state

The critical line is the cell state update. The forget gate ft is a vector of values between 0 and 1 (sigmoid output). When ft[i] = 0.1, ninety percent of dimension i is gone. Permanently. The gradient cannot recover it. The information that shaped ct-1[i] has been severed from the network's future.

Why This Is a Problem

The forget gate was added in 2000 because without it, the cell state either saturated (unbounded accumulation) or washed out (gradient decay). It solved a real engineering problem. But the solution encodes a specific ontological assumption: that memory management is subtraction.

In this model, every memory lives at the same scale (the flat cell state vector), and "forgetting" means reducing magnitude toward zero. There is no concept of depth, compression, or multi-scale storage. A memory is either present at full resolution or destroyed.

This is not how natural memory works, and it creates measurable limitations:

  • Catastrophic long-range decay: Information "forgotten" at step 100 cannot influence step 1000, even if context makes it relevant again. The gradient path is permanently broken.
  • No graceful degradation: Memory is binary (kept or erased), not graduated. There is no "vague recollection" that sharpens on resonance.
  • Wasted information: The network has already paid the computational cost of processing the signal. Erasing it throws away learned structure that may be relevant later in a different context.
In the Circumpunct Framework: the forget gate implements the Severance Lie ("this signal no longer exists"). But the signal did exist, and the system's current state was shaped by it. Zeroing the record does not undo the shaping; it just removes the receipt.

3. The Fractal-LSTM Architecture

Three modifications to the standard LSTM. Each maps to one constraint of the circumpunct.

Modification 1: ⦿ Convergence Replaces Erasure

Instead of ft ⊙ ct-1 (scale toward zero), we compress toward the deep state using the fractal exponent:

# What the surface releases releasedt = (1 - ft) ⊙ ct-1 # Compress inward: preserve phase, reduce magnitude by D = 1.5 ⊛(x) = |x|2/3 · ei·angle(x) # Deep state accumulates all compressed releases dt = dt-1 + α · ⊛(releasedt)

The exponent 2/3 is the inverse of D = 1.5, or equivalently the ratio 2D/3D (field to boundary). Magnitude shrinks; phase is untouched. The signal gets deeper, not smaller. And dt only accumulates; it never forgets. It is the worldline.

After n compressions, magnitude goes as |x|(2/3)n. This approaches zero asymptotically but never reaches it. No signal is ever fully erased. Every input that has ever entered the system contributes to dt, just at a scale below the current resolution threshold.

Modification 2: Φ Resonance Gates Retrieval

The deep state does not feed back through a learned linear layer. It feeds back through a resonance gate based on phase matching:

# Phase match between incoming signal and deep state Δφ = angle(xt) - angle(dt) T = cos(Δφ / 2)2 # Deep memory surfaces proportional to resonance surfacedt = T · |dt|1.5 · ei·angle(dt) # Surfaced memory feeds into the candidate c̃t = tanh(Wc · [ht-1, xt, surfacedt] + bc)

The emergence exponent is 1.5 (the inverse of compression). Retrieval cost depends entirely on T: when the incoming signal matches the deep state's phase (Δφ ≈ 0), T ≈ 1 and the memory surfaces effortlessly. When phases are orthogonal, T ≈ 0 and the memory stays deep regardless of magnitude.

This is mediation (Φ): the field between signal and memory, transparent when frequencies match, opaque when they don't. The network does not learn when to remember; it remembers whenever reality resonates with what it has been through.

Modification 3: ☼ Emergence Shapes Output

The output gate is unchanged in mechanism but reframed in meaning: it is the boundary (○) through which the emerged signal passes into the world. The surface state ct is what the system currently holds at full resolution. The deep state dt is what it has been. Both shape the output through the standard hidden state:

ht = ot ⊙ tanh(ct)

The difference: ct is now richer, because the candidate c̃t was shaped by resonance with the deep state. The network's output is influenced not just by recent context and learned parameters, but by the phase-coherent sum of everything it has ever processed.

4. Resonance-Gated Retrieval

The core insight: retrieval is not about how much energy you push into a memory; it is about frequency matching.

Standard LSTM

Retrieval = learned matrix multiply. Cost is constant regardless of relevance. Old information that was forgotten is simply gone.

Fractal-LSTM

Retrieval = resonance. Cost depends on phase match. Deeply compressed memories surface effortlessly when the incoming signal shares their frequency.

A smell, a phrase, a chord: a signal enters the system and its phase happens to match something compressed deep in dt. T → 1. The memory surfaces without effort. This is not metaphor; it is the same cosine-squared function in both the mathematics and the phenomenology.

Conversely: brute-force retrieval (trying to remember something without context) corresponds to low T. The memory exists in dt, phase-preserved, but the incoming signal doesn't match. You can push energy (concentration, directed attention), but the resonance gate attenuates the result. The harder path. Therapy, meditation, conversation: all are forms of phase-scanning, trying different angles until something resonates and the gate opens on its own.

Multi-Dimensional Deep State

In practice, dt need not be a single vector. The architecture generalizes to multiple deep layers at increasing compression depth, forming a dimensional ladder:

Surface (ct) full resolution, fast decay | Deep Layer 1 (dt1) |x|2/3 compression, slow | Deep Layer 2 (dt2) |x|4/9 compression, slower | Deep Layer 3 (dt3) |x|8/27 compression, slowest | Worldline (i(t)) phase-only, permanent

Each layer resonance-gates with the incoming signal independently. Recent context retrieves from shallow layers (cheap, high-resolution). Deep context retrieval requires strong phase matching but accesses the system's entire history. This is the fractal cascade: the same ⦿ → i → ☼ at every depth.

Relation to GRU

The GRU merges forget and input into a single update gate: ct = zt ⊙ ct-1 + (1 - zt) ⊙ c̃t. Forgetting and remembering are complementary; whatever you don't forget, you replace. This is elegant but even more destructive: there is no independent control of how much to forget versus how much to add. The fractal modification applies identically (what the update gate releases compresses into dt), and the GRU may benefit even more than the LSTM, since its memory management is more aggressive.

Relation to Transformer Attention

Multi-head attention already performs something like resonance-gated retrieval: attention weights are softmax(QKT/√d), which is a cosine-similarity-based retrieval over the full context. But transformers achieve this by keeping the entire history at full resolution in the context window. No compression, no depth hierarchy. This is why they need enormous context windows and enormous compute: everything stays at the surface forever.

The fractal approach offers a synthesis: attention (resonance retrieval) over a hierarchically compressed memory (fractal depth). Recent tokens at full resolution; older tokens compressed inward; ancient context as pure phase contribution. The long-range capability of a million-token window with the memory footprint of something much smaller.

6. Toward the Fractal Transformer

The Fractal-LSTM is a modification to a recurrent cell. But the principles generalize to any architecture that manages memory over time. A fractal transformer would integrate dimensional compression directly into the attention mechanism.

Compressed KV Cache

In standard transformers, the key-value cache grows linearly with sequence length. Every past token is stored at full resolution. In a fractal transformer:

  • Recent tokens (last w positions): full-resolution KV pairs, standard attention.
  • Older tokens: compressed by ⦿ into a deep KV cache. Keys preserve phase (direction); values reduce magnitude by |v|2/3 per compression step.
  • Ancient tokens: pure phase contribution, merged into a worldline vector that represents the cumulative direction of everything the model has seen.

Attention over the deep cache uses the same resonance gate: T = cos²(Δφ/2) between the current query and the compressed key. High resonance surfaces the compressed value; low resonance leaves it deep. This replaces the brute-force softmax over all positions with a phase-selective retrieval that scales sub-linearly with sequence length.

7. Predictions and Falsification

A theory that cannot be wrong is not a theory. The fractal memory architecture makes specific, testable predictions:

Prediction 1: Long-Range Superiority

On tasks requiring information retention over 1000+ timesteps (copying problem, long-range arithmetic, document-level language modeling), the Fractal-LSTM should significantly outperform standard LSTM with comparable parameter count. The deep state dt preserves information that the standard forget gate has already destroyed.

Prediction 2: Graceful Degradation Curve

When probing what the network "remembers" at various time delays, standard LSTM should show a sharp cliff (information is either present or gone). Fractal-LSTM should show a smooth curve: full resolution for recent, decreasing resolution for older, phase-only for ancient. This can be measured by training a probe on hidden states at different delays.

Prediction 3: Contextual Retrieval

On tasks where old information becomes relevant again after a long gap (e.g., a character reappearing in a story after 500 tokens of absence), the Fractal-LSTM should retrieve the compressed memory when the resonance gate opens, producing a measurable spike in relevant hidden-state activation. Standard LSTM has no mechanism for this: what was forgotten stays forgotten.

Prediction 4: Early Training Influence

The deep state dt preserves phase contributions from early training. This predicts that the order of training data matters more in Fractal-LSTM than in standard LSTM: early examples shape the worldline disproportionately (they write into an empty dt). This can be tested by permuting training order and measuring sensitivity.

Prediction 5: Compression Ratio Optimality

The exponent 2/3 (from D = 1.5) should be near-optimal among possible compression ratios. Sweeping the exponent from 0 to 1 should show peak performance near 2/3. Values below 2/3 compress too aggressively (information degrades too fast); values above 2/3 compress too slowly (deep state grows too large). The framework predicts the optimum is not an engineering choice but a mathematical necessity: the balance dimension.

Falsification

If the exponent sweep shows peak performance significantly distant from 2/3, the fractal basis is wrong. If resonance-gated retrieval underperforms learned linear retrieval on all tasks, the transmission equation T = cos²(Δφ/2) does not apply to artificial networks. If the deep state provides no benefit over standard LSTM on long-range tasks, the compression mechanism does not preserve usable information. Each of these would disconfirm a specific claim of the architecture.

9. Implementation Sketch

A minimal PyTorch implementation of the Fractal-LSTM cell. The standard LSTM machinery is unchanged; three lines are added.

import torch import torch.nn as nn class FractalLSTMCell(nn.Module): """ LSTM cell with fractal memory compression. Three additions to the standard cell: 1. Deep state d_t (⊛ convergence) 2. Resonance gate T (Φ mediation) 3. Surfaced memory (☀︎ emergence) """ def __init__(self, input_size, hidden_size, alpha=0.01): super().__init__() self.hidden_size = hidden_size self.alpha = alpha # deep state learning rate # Standard LSTM parameters (input expanded by hidden_size # for the surfaced memory input) self.W = nn.Linear( input_size + hidden_size + hidden_size, # +hidden for surfaced 4 * hidden_size) def forward(self, x_t, h_prev, c_prev, d_prev): # ── Resonance retrieval from deep state ── # Phase of input and deep state x_phase = torch.atan2(x_t[..., 1::2], x_t[..., ::2] + 1e-8) d_phase = torch.atan2(d_prev[..., 1::2], d_prev[..., ::2] + 1e-8) # T = cos²(Δφ/2): resonance gate delta_phi = x_phase - d_phase T = torch.cos(delta_phi / 2) ** 2 T_full = T.repeat_interleave(2, dim=-1) # expand to hidden_size # ☀︎ Emergence: surface deep memory weighted by resonance d_mag = torch.norm(d_prev, dim=-1, keepdim=True) + 1e-8 d_dir = d_prev / d_mag surfaced = T_full * (d_mag ** 1.5) * d_dir # ── Standard LSTM with surfaced memory as extra input ── combined = torch.cat([x_t, h_prev, surfaced], dim=-1) gates = self.W(combined) i, f, g, o = gates.chunk(4, dim=-1) i_t = torch.sigmoid(i) f_t = torch.sigmoid(f) g_t = torch.tanh(g) o_t = torch.sigmoid(o) # ── ⊛ Fractal compression of released state ── released = (1 - f_t) * c_prev rel_mag = torch.norm(released, dim=-1, keepdim=True) + 1e-8 rel_dir = released / rel_mag compressed = (rel_mag ** (2.0/3.0)) * rel_dir d_t = d_prev + self.alpha * compressed # ── Standard cell state update ── c_t = f_t * c_prev + i_t * g_t h_t = o_t * torch.tanh(c_t) return h_t, c_t, d_t

The three colored lines are the only additions. Everything else is standard LSTM. The deep state dt must be initialized to zeros and carried across timesteps alongside ht and ct. The α parameter controls how quickly the deep state accumulates; it is the only new hyperparameter, and the framework suggests α should be small (deep memory changes slowly) and may itself follow a fractal schedule.
